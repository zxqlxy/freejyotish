- Got a new astrology glyph font for FJ. Started integrating it into the program. Wrote the constants necessary to using this font to FJConstants file.
- Began creating new, simplified NativeInfo2 persistence mech. This is now the third persistence strategy for FreeJyotish. An attempt to get very simple. The XML mech works fine, but is bloated. The java serialization mechanism makes it so the slightest change in the code invalidates all the saved charts. So this is an attempt to make a very direct and simple "|" delimited file that saves only the birth data for each chart. This will mean re-working (again) the entire ChartCenter to adapt it back to saving NativeInfo2 objects instead of Chart objects. When loading saved charts, all chart data will be recalculated.

26 APR 03
- Emails back and forth with Ken regarding new calculator. He is recommending using a flexible form of Java object that I've never seen before. Basically, rather than defining variables and then having the standard get/set methods, this type of object is basically a Hashtable on a stick. That way it can hold as many of whatever kind of variables you choose, and as long as you know the names you stored them under, you can get them back out and cast them correctly. Has only on get/set pair, which is really a get/put pair. Very interestink...
